# Node: write-seed

Write `$SESSION_DIR/SEED.md`. Sole writer of the artifact.

## Inputs

- `mode`, `session_dir`, `session_ts`, `project_url`, `seed_text` from walker state
- The synthesized seed text from `synthesize` (in-memory or `seed_text` if persisted)
- The "context discussed" summary (what was explored, alternatives rejected, key constraints)

```bash
STATE="$SESSION_DIR/.walk-state.json"
MODE=$(scripts/walk.sh get --state "$STATE" --key mode)
SESSION_DIR=$(scripts/walk.sh get --state "$STATE" --key session_dir)
SESSION_TS=$(scripts/walk.sh get --state "$STATE" --key session_ts)
PROJECT_URL=$(scripts/walk.sh get --state "$STATE" --key project_url)
```

## Steps

Write `$SESSION_DIR/SEED.md` from this template:

```markdown
# Seed: <topic>

[The 2-3 paragraph refined seed from synthesize.]

## Context discussed

[Brief summary — what was explored, alternatives rejected with reasons,
key constraints surfaced. Bullets are fine here.]

## Source signals

[Mode-specific. See blocks below.]

---

*Generated by `/sprint-seed` on <YYYY-MM-DD>.*
*Run `/sprint-plan <path-to-this-file>` to plan.*
```

### Source signals — Linear mode

```markdown
## Source signals

- **Project**: <project URL>
- **Milestones**: <complete count> complete, <in-flight> in flight, <untouched> untouched
- **Gap this seed fills**: <one-line synthesis>
```

### Source signals — Repo mode

Pointers to past sprints/retros that informed this seed, with full session paths so `/sprint-plan` can reference them:

```markdown
## Source signals

- Sprint 2026-04-12T... deferred X — `~/Reports/<org>/<repo>/sprints/<TS>/SPRINT.md`
- Retro 007 flagged Y as recurring — `~/Reports/<org>/<repo>/sprints/<TS>/RETRO.md`
- Natural sequel to <topic> — `~/Reports/<org>/<repo>/sprints/<TS>/SPRINT.md`
```

### Source signals — Seed mode

Omit the `## Source signals` section entirely. Orientation was light; there are no source-system snapshots to pin.

## Topic

Derive `<topic>` from the discussion's converged scope. Short noun phrase, completion-framed where natural ("Cache warmup for python images", not "Cache warmup work"). Avoid the word "sprint" in the title.

## Outputs

- `$SESSION_DIR/SEED.md`

## Outgoing edges

- → `handoff` (always — single outgoing edge)

Record the transition:

```bash
scripts/walk.sh transition --state "$STATE" --from write_seed --to handoff
```

## Failure modes

- `$SESSION_DIR` missing somehow → recreate it (`mkdir -p`) and write. Should never happen given `init`.
- Disk full → surface error verbatim, route to terminal anyway via handoff (handoff will note the failure).
